Content Analysis

Technical Description

My content page covers "Water Quality Monitoring" and uses the shared template header and footer. The page structure follows the same semantic HTML5 pattern as the rest of the site, with a <main> element for the article body and supporting sections. I used <figure> and <figcaption> elements to pair images with their descriptions.

The CSS uses the same colour variables defined in style.css (e.g. var(--primary), var(--secondary)) so the page looks consistent with the rest of the site. Key information boxes use background, border-radius, and padding to create visual separation from the running text.

Accessibility

All images have descriptive alt text, and the heading hierarchy is sequential (<h1> → <h2> → <h3>). Links to external sources open in new tabs and are labelled clearly.

Challenge

The biggest challenge I faced was getting the SVG progress ring to animate smoothly. I initially tried using a simple <div> with border-radius: 50% and a border, but that did not allow partial filling. I learned about the stroke-dasharray and stroke-dashoffset SVG properties, which let me control exactly how much of the circle is drawn.

The splash page countdown was also tricky. I first relied only on the <meta refresh> tag, but I wanted a visible countdown for users. Adding the JavaScript setInterval was straightforward, but I had to make sure both the meta redirect and the JavaScript redirect didn't conflict. I solved this by having JavaScript clear the interval and redirect before the meta tag fires.

I also struggled with making the AIS grid responsive. On mobile, a 3-column grid is too narrow. I used a media query to switch to grid-template-columns: 1fr on screens below 768px, which solved the issue. This taught me the importance of testing at different screen widths early in development.

Compliance

My pages comply with the Janet Acceptable Use Policy. All content is educational and relates to SDG 6: Clean Water and Sanitation. No personal data is processed — the Splash page simply displays an animated countdown and redirects to the Home page. The AIS simulator runs entirely in the browser using client-side JavaScript; no server requests are made and no data is stored or transmitted.

All images are either project-generated or sourced from royalty-free platforms and properly attributed. The pages do not contain advertisements, tracking scripts, or third-party analytics. External links (such as to lecture notes) open in new tabs and are clearly labelled.

I validated my HTML and CSS using the W3C validators. Results are documented on the validation page with screenshots.

Accessibility

For the Splash screen and AIS Simulator, I focused on keyboard accessibility and aria-labels. The countdown timer includes a hidden live-region for screen readers, and the simulator's interactive grid items provide text feedback when clicked. I also ensured high contrast for the status indicators to assist users with color-vision deficiencies.
