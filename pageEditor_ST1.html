Home Analysis

Technical Description

The Home page serves as the main landing page for Rippl and was built using a combination of semantic HTML5 elements and embedded CSS. I used the <header>, <main>, <section>, and <footer> tags to create a clear document structure. The hero section uses a full-width background image inside a <div> with position: absolute and object-fit: cover so that the image fills the viewport without distortion. A gradient overlay (linear-gradient) sits on top to improve text readability.

The statistics section uses flexbox (display: flex) to arrange a header row and card components. The large "Aquatic Integrity System" card has a semi-transparent background image achieved with opacity: 0.2 on the <img> element, while the text content is given a higher z-index to remain on top. The "Pillars of Sustainability" section organises three feature cards in a column layout using flex-direction: column with a gap of 2rem between items.

For the "Explore Our Research" gallery at the bottom, I used flex-wrap: wrap so the cards reflow on smaller screens. Each research card has a :hover effect with transform: translateY(-5px) and a gradient overlay created with linear-gradient(to top, ...) to display text on top of images.

Accessibility

Every image on the Home page has a descriptive alt attribute — for example, the hero image has alt="Deep blue ocean water with sunrays piercing through the surface". The navigation uses an aria-label="Toggle navigation" on the mobile hamburger button. I made sure the colour contrast between white text and the dark blue background (#001f3f) meets WCAG AA requirements. All interactive elements such as links and buttons are keyboard-accessible by default because I used native <a> and <button> elements rather than styling <div> tags.

Gallery Analysis

Technical Description

The Gallery page (titled "The Fluid Archive") displays six image cards in a responsive grid. I used display: flex with flex-wrap: wrap on the .gallery-grid container, giving each card flex: 1 1 calc(33.333% - 1rem) so they arrange into three columns on wide screens and collapse to fewer columns on smaller devices via a min-width: 280px constraint.

Each gallery item has a hover overlay implemented with two layers: a .gallery-info-overlay that appears on hover via JavaScript event listeners (mouseenter / mouseleave). When hovered, a CSS class .hovered is added, which transitions the overlay from opacity: 0 to opacity: 1. The overlay has a linear-gradient(to top, ...) background so the label text stays readable.

The modal popup is built with a .modal-backdrop container that covers the full viewport using position: fixed and inset: 0. JavaScript functions openModal(index) and closeModal() control showing and hiding the modal by toggling an .active class. A keyboard listener closes the modal when the user presses Escape (e.key === 'Escape').

Accessibility

All gallery images carry descriptive alt text such as alt="Advanced water sanitation facility". The modal close button uses aria-label="Close modal". The gallery items are clickable via their onclick attribute and the modal can be dismissed with Escape, ensuring keyboard users can navigate without difficulty. I also ensured colour contrast is sufficient on the hover overlay text (white on a dark gradient).

Content Analysis

Technical Description

My content page focuses on "Water Conservation Strategies" and follows the shared template structure. The page layout uses semantic tags: <article> for the main written content and <aside> for supplementary statistics. I used CSS border-radius and box-shadow on card components to separate visual sections.

The page links back to the Home page within its summary section. Images are set to width: 100% with object-fit: cover so they scale proportionally. I used an ordered list (<ol>) for step-by-step conservation tips and an unordered list (<ul>) for quick-reference facts, keeping the document semantically meaningful.

Accessibility

Heading hierarchy follows a logical flow from <h1> to <h3>, which aids screen reader navigation. All images have relevant alt text. Links are styled visually with an underline and colour contrast to distinguish them from body text.

Challenge

One of the biggest challenges I faced was making the hero image on the Home page look good on all screen sizes. Initially the image appeared stretched on very wide monitors. I fixed this by using object-fit: cover combined with a set min-height, which crops the image rather than stretching it. This taught me that responsive images require more than just setting width: 100%.

Another issue was the gallery hover overlay not appearing on mobile because there is no hover state on touchscreens. I learned that JavaScript event listeners (mouseenter/mouseleave) are more reliable than CSS :hover alone when you need to support both desktop and mobile interactions.

I also had some trouble keeping the navigation bar consistent across all pages. Every time I changed a link, I had to update it on every HTML file manually. This taught me the importance of planning the site structure carefully before starting development, and made me appreciate why frameworks use reusable components.

Compliance

All pages I developed follow the Janet Acceptable Use Policy guidelines. The content is original and created solely for educational purposes. No copyrighted material is used without proper attribution — all images are either generated or sourced from royalty-free providers and credited in the References section.

The pages do not contain any offensive, discriminatory, or misleading content. Personal data is not collected or stored; the feedback form is a front-end demonstration only and does not transmit data to a server. All external links open in new tabs using target="_blank" where appropriate, so users are aware they are leaving the site.

I have validated my HTML using the W3C Markup Validation Service .The validation results and snapshots are documented on my validation page.

Accessibility

To ensure WCAG 2.1 compliance, I implemented a logical heading hierarchy and high-contrast color schemes across the Home and Gallery pages. All informative images include descriptive alt text, and interactive elements like buttons have unique IDs for better screen reader navigation. I also ensured that the navigation menu is accessible via keyboard-only input using proper tab indices.
