Content Analysis

Technical Description

My content page covers "Community Sanitation Programs" and describes how local communities can improve hygiene through grassroots initiatives. The page uses the shared template and follows the same heading hierarchy as the other content pages. The layout uses basic CSS properties such as padding, margin, and max-width to keep the text readable at around 70 characters per line.

I used <blockquote> elements for featured quotes and <table> elements with proper <thead> and <tbody> for statistical data. Images are displayed with border-radius and a subtle box-shadow for visual consistency with the rest of the site.

Accessibility

The heading structure is logical and sequential. All images have meaningful alt text. The table uses <th> elements for header cells, helping assistive technologies understand the data structure. Links are underlined and use the site's secondary colour for visual distinction.

Challenge

The hardest part of building the Feedback page was JavaScript form validation. At first, I used only the browser's built-in validation with the required attribute, but this did not let me customise error messages or their placement. I then wrote my own validateForm() function that manually checks each field and shows custom error messages below the inputs. I learned that e.preventDefault() is essential to stop the form from submitting before validation is complete.

Styling the scrollable programme cards area was also challenging. I wanted a fixed-height container with a scrollbar, but the default scrollbar looked unattractive. I used ::-webkit-scrollbar pseudo-elements to customise the scrollbar width and thumb colour, though I learned this only works on WebKit-based browsers.

On the Team page, getting all four cards to be the same height in a flex row was tricky because the descriptions varied in length. I solved this with align-items: stretch (the default in flexbox), which forces all items in a row to match the tallest card's height.

Compliance

All pages I developed comply with the Janet Acceptable Use Policy. The feedback form is a purely client-side demonstration — it does not submit data to any server, store cookies, or collect personally identifiable information. The success message is displayed only on-screen and no data leaves the user's browser.

The Team page uses only original or royalty-free images and all team member names and details are fictional, created solely for the purposes of this coursework. Content on the site promotes SDG 6 educational awareness and contains no offensive, discriminatory, or commercial material.

I validated all my pages using the W3C Markup Validation Service and the W3C CSS Validation Service. Screenshots of the results can be found on the validation page.

Accessibility

For the Feedback and Team pages, I focused on form accessibility and clean navigation. The feedback inputs use correctly associated <label> tags, and error messages are aria-described to the relevant inputs. The team cards use semantic <h3> tags for member names, ensuring screen readers clearly announce the team member descriptions.
